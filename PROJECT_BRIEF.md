# 英文模板站项目计划工程书（执行定稿版 v5）

Updated: 2026-04-04 23:14 Asia/Shanghai
Status: Execution locked v5

## 1. 项目名称
英文简历模板站验证项目（真实搜索需求入口 + AI/ML 差异化增强）

## 2. 项目一句话定义
从已有真实英文简历搜索需求切入，不直接追 AI 职位名关键词；通过 `AI/ML 项目改写能力 + no experience 转化路径` 做差异化，先验证 `收录 + impressions + query 质量 + 流量导向 pillar 的能力`，再决定是否放大。

## 3. 当前总原则
- 不直接追 AI 岗位名关键词
- 用现有真实搜索需求做入口
- 用 AI/ML 差异化增强切口
- 先验证 query 质量，再谈收入
- 先做一个站，再考虑放大
- 主关键词避免使用 `ml` 缩写

## 4. 当前目标用户
- 美国 / 英语国家用户
- fresh graduates looking for full-time jobs
- no-experience / entry-level 求职人群
- 有 software / computer science / data 相关背景，但想往 AI/ML 岗位靠的人

## 5. 当前真实痛点定义
- 没有 AI/ML 工作经验，简历怎么写得像能投 ML 岗位
- 只有普通软件/CS背景，怎么改写成更接近 AI/ML 候选人
- 没有正式经验，项目怎么写、技能怎么摆、JD 怎么映射

## 6. 最终 round 1 词簇

### Pillar
- `entry level machine learning engineer resume`

### Support
- `software engineer resume no experience`
- `entry level data science resume`
- `how to write machine learning resume without experience`

### Longtail
- `machine learning projects for resume beginner`

## 7. 明确删除/不用的词
- `ai_application_engineer_resume`
- `fresh_graduate_ai_engineer_resume`
- `computer_science_resume_ai_projects`
- `ml_projects_for_resume_examples`
- `how_to_write_ml_resume_without_experience`

原因：
- AI title keywords 过虚
- `ml` 缩写带来歧义和不稳定 SERP 映射
- builder-generated phrasing 风险高

## 9. 页面职责划分

### Pillar
负责回答：
- 没有 AI/ML 正式经历的人，怎么做出一份可投递 ML engineer 岗位的简历

### software-engineer-resume-no-experience
负责：
- 吃泛软件简历需求
- 把用户往 ML/AI 路径引
- 不做核心转化，只做入口

### entry-level-data-science-resume
负责：
- 提供更自然、更低竞争的相邻入口
- 承接 Data Science 相邻需求，再引流至 ML 路径

### how-to-write-machine-learning-resume-without-experience
负责：
- 讲清“没经验怎么往 ML 简历靠”
- 偏路径说明和桥接逻辑

### machine-learning-projects-for-resume
负责：
- 讲清项目怎么挑、怎么写、怎么映射 JD
- 偏方法页

## 10. 页面必须具备的模块
- copyable resume template
- real resume example
- AI/ML project breakdown
- JD to resume mapping
- common mistakes

### 核心差异化卖点
- show how to convert non-AI background into ML resume
- map projects to real job descriptions
- provide copy-ready project bullets

## 11. 内链逻辑
- longtail page -> bridge pages
- bridge pages -> pillar
- traffic entry page -> ML path
- pillar <-> all supporting pages

## 12. 当前验证目标
### Success Signals
- pages indexed
- impressions increasing
- queries contain:
  - `machine learning resume`
  - `software engineer resume`
  - `no experience`
  - `data science resume`

### Failure Signals
- impressions but mostly irrelevant queries
- no ML / software-engineer / data-science related useful queries
- traffic not flowing toward the pillar path

## 13. 当前执行规则
- do_not_scale_before_query_validation
- do_not_chase_ai_keywords_directly
- attach_ai_signal_to_existing_search_demand
- prioritize_user_search_language_over_builder_logic
- avoid_ml_abbreviation_in_primary_keywords

## 14. 当前结论
当前版本已经足够进入 SERP 实查和页面结构设计阶段。
接下来重点不是再改关键词逻辑，而是验证这组词是否能打，并把页面结构做出足够强的差异化。
